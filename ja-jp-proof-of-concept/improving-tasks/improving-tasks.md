# タスクを改善します

## 序章

このラボでは、値の改善されたリストを利用することにより、タスクフォームページを改善する方法を学びます。

推定時間：15分

### 目的

- 値のプロジェクトリストを改善します
- タスクフォームページを更新します

### あなたは何が必要ですか？

- Oracle Cloudの有料アカウント、Livelabsアカウント、または無料トライアル。
- 頂点アプリケーション

## タスク1：値のプロジェクトリストの改善。

ランタイム環境でタスクに移動し、任意のレコードの編集アイコンをクリックして、タスクフォームページを表示します。

![](images/project-list.png " ")

プロジェクトに関連する追加の詳細を確認することは有益です。

1. ランタイムアプリケーションから、ページデザイナーのタスクフォームページに移動します。

   Apex App Builderからこのアプリを実行したことを考えると、画面の下部に開発者ツールバーが表示されます。

   開発者ツールバーでは、[8ページを編集] **をクリックします。

   ![](images/dev-toolbar.png " ")

   または、適切なブラウザタブまたはウィンドウを選択して、ブラウザのApexアプリビルダータブに手動でナビゲートすることもできます。

   ![](images/alt-app-builder.png " ")

   これで、**ページ8：タスク**のロードされたページデザイナーにいる必要があります。


2. プロジェクトに使用されている値の現在のリストを確認する必要があります。

   ページデザイナー内のレンダリングツリー（左ペイン）で、[** p8 \ _project_id **]をクリックします。

   プロパティエディター（右ペイン）では、フィルターに**リスト**を入力します。

   プロパティエディター（右ペイン）で、**値のリストの矢印をクリックします**。

   ![](images/go-lov.png " ")

   *{注：値のリスト>タイプは_sharedコンポーネント_であるため、適切な共有コンポーネントを編集する必要があります。}*

3. 左ペインは[共有コンポーネント]タブに変更され、適切な値のリストが強調表示されます。

   プロパティエディター（右ペイン）で、[コンポーネント] **の編集**をクリックします。

   ![](images/go-component.png " ")

4. 次に、いくつかの追加の列を追加します。

   値のリスト：hol_projects.nameページ、[**列]を選択します**をクリックします。

   [追加の表示列の選択]ダイアログで、次の列をダブルクリックします。
    -   スターテス
    -   completed_date
    -   説明

   [**更新]をクリックします**

   ![](images/add-columns.png " ")

   [変更を適用] **をクリックします**。

5. 追加のディスプレイ値を表示するには、選択リストの代わりに、ページアイテムをポップアップLOVに更新する必要があります。

   ページデザイナー内で、レイアウト（中央のペイン）で、** p8 \ _project_id **をクリックします

   プロパティエディター（右ペイン）で、識別>タイプの場合、[**ポップアップLOV **を選択します。

   ページデザイナーツールバーで、**保存**をクリックします。

   ![](images/set-popup-lov.png " ")

   *{注：_save_の代わりに_runとsave_をクリックすると、ページは保存されますが、このページはモーダルページであり、ページデザイナーから直接実行できないため、警告が表示されます。}*

   ![](images/run-save.png " ")

6. 改良されたプロジェクトアイテムを確認する時間です！

   [ランタイム環境]タブまたはブラウザに移動します。

   ![](images/new-proj-lov.png " ")

   *{注：列のサイズを調整したり、ポップアップLOVに表示されている列をソートすることもできます。}*

## タスク2：値のカスケードリストの定義

マイルストーンの選択リストを確認すると、すべてのマイルストーンがリストされていることに気付くでしょう。

1. 開発環境に戻ります。
2. ページデザイナー内で、8ページが選択され、ページデザイナーツールバーで、[共有されたコンポーネント**]をクリックします。

   ![](images/go-shared.png " ")

   または、アプリケーションのホームページから、**共有コンポーネント**をクリックします。

   ![](images/alt-go-shared.png " ")

3. 共有コンポーネントから、他のコンポーネントの下で、[値のリスト] **をクリックします。

   ![](images/go-shared-lov.png " ")

4. [値]ページのリストから、**コピー**をクリックします。

   [値のコピーリスト]ダイアログに次のものを入力します。
    - 値のリストをコピー -  ** hol_milestones.name -dynamic **を選択
    - 値の新しいリスト名-Enter ** hol_milestones.Name Page 8 **

   **コピー**をクリックします。

   ![](images/copy-lov.png " ")

5. これで、新しいLOVをカスタマイズする必要があります。

   値のリストのレポートで、** hol_milestones.nameページ8 **をクリックします

   ![](images/go-p8-lov.png " ")

6. 値の編集ページのリストで、句は以下を入力します。

   ```
   <copy> project_id = :P8_PROJECT_ID
   </copy>
   ```

   *{注：これは、プロジェクトアイテムのために選択された現在の値_p8 \ _project \ _id _。}*に基づいて返されるマイルストーンを制限します。

   [変更を適用] **をクリックします**。

   ![](images/set-p8-lov.png " ")

## タスク3：タスクフォームページの更新

1. 次に、8ページのアイテムに新しいLovを接続する必要があります。

   [値]ページのリストから、ツールバーで、[8ページの編集] **をクリックします。

   ![](images/go-page8.png " ")

2. ページデザイナーでは、左ペインで**レンダリング**アイコンをクリックします。

   プロパティエディター（右ペイン）に、以下を入力してください。
    - 値のリスト-Select ** hol_milestones.Name Page 8 **
    - cascading lov parent item（s） - 選択** p8 \ _project_id **を選択します

   **保存**をクリックします。

   ![](images/set-milestone-lov.png " ")

3. マイルストーンアイテムをテストする時間。


   [ランタイム環境]タブまたはブラウザに移動します。

   ![](images/new-milestone-lov.png " ")

   *{注：マイルストーンアイテムの名前ではなく、定期的にリストされている数字があります。

   *このサンプルデータの問題を修正するには、選択したプロジェクトに関連するマイルストーンを表すため、利用可能なリストから名前の1つを選択するだけです。}*

4. 選択されたプロジェクトがない場合に何が起こるかを見る最も簡単な方法は、新しいタスクを作成することです。

   ランタイム環境では、モーダルページを閉じてタスクレポートページに戻ります。

   ![](images/create-task.png " ")

   *{注：プロジェクトが選択されていないため、マイルストーンがリストされていないはずです。}*

## **まとめ**

さまざまな種類の値のリストを使用して、追加の列を表示する方法、およびカスケードLOVを定義する方法を知っています。
